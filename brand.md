## App Name

The app name is **requiem**. Always lowercase. Render in small caps (`font-variant: small-caps`) in display contexts (wordmark, headings).

The App Store listing uses **requiem.http** to satisfy uniqueness requirements. This is not the display name.

## Terminology

Use **HTTP** exclusively when referring to the protocol, requests, or the app's domain. Do not use **REST** as a synonym or play on words — it overloads the term and muddies the message.

The only acceptable use of **.rest** is alongside **.http** when listing supported file extensions (e.g. ".http and .rest files").

The domain **requiem.rest** uses the `.rest` TLD because it is a popular namespace for HTTP clients. Do not draw attention to the domain name as a pun or wordplay.

## Tagline

Site tagline: **.http for Shortcuts**. The App Store subtitle field is currently "Build Complete HTTP Requests".

## App Store Promotion

Do not add a Smart App Banner (`<meta name="apple-itunes-app">`) to requiem.rest. Install banners are annoying on product pages where the user is already looking at the app. Use the standard badge button instead.

req.to has an install banner because the app is genuinely part of that product's workflow.

## Voice

Technical, terse, confident. No exclamation marks. No marketing superlatives. Let the feature speak.
